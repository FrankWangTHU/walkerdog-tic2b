# 🐕 项目简介

**DogWalker（狗溜人）** 是一个基于 Unitree Go1 四足机器人、Jetson Nano 和 2D LiDAR 的室内自主导航与情感交互系统。

它的核心目标：

* 解决“久坐不动”的生理惰性
* 解决“社恐不敢见导师/老板”的心理阻碍
* 让机器人主动来“溜”你，而不是被你溜

它不是宠物，而是你的——
**健康教官 & 社交辅助伙伴。**

---

# 💡 为什么做这个？ (Problem & Value)

### 🩻 1. 生理惰性：久坐成疾

大学生和程序员平均每天久坐 8 小时以上。
但光靠自觉往往不够：
**“再写五分钟就去走走…” → 半天过去了。**

### 🧠 2. 社交畏难：需要“外力推动”

找导师签字、找老板沟通——
很多人会：

* 不敢去
* 不想去
* 再拖拖
* 心里发慌

**心理成本比物理距离大得多。**

# 🌟 核心场景与用户故事

## 🟦 场景一：健康模式（Health Pacer）

**用户故事：**
“我已经坐了 3 小时了，腰酸，但我就是不想动。”

**流程：**

1. 系统检测“久坐”时间或用户主动点击“溜我一下”
2. 机器狗导航到工位
3. 播放轻松语音：“动一动，跟我去茶水间～”
4. 机器狗领跑前往茶水间
5. 抵达后给予正向反馈

---

## 🟥 场景二：勇气模式（Boss Mode）🔥 *（核心亮点）*

**用户故事：**
“我必须找导师签字，但我非常害怕。”

**流程：**

1. 用户点击 “带我去导师办公室”
2. 机器狗导航到工位，进入“战备状态”（BGM/语音）
3. 机器狗领跑，用户跟随
4. 掉队检测：

   * 当人因害怕停下
   * 狗立即停下 → 转身 → 语音鼓励/激将：
     **“别怂！你可以的！”**
5. 到达导师办公室 → 狗趴下示意“已送达”
6. 用户只剩下面对导师，不再逃避

# 🧩 系统功能模块

---

## 1. 导航模块

**优先级：P0（最高）**

### 技术栈：

* 2D LiDAR（思岚 A1）
* SLAM：gmapping
* 定位：AMCL
* 路径规划：move_base（局部推荐 TEB）

### 需求：

* 室内建图（工位/走廊/障碍物）
* 自主定位
* 全局/局部规划
* 动态避障（突然出现的人或椅子）

---

## 2. 交互与逻辑控制模块

**优先级：P0**

### Web 控制面板包括：

* “去茶水间”
* “去导师办公室”
* “回充”

### 核心算法：领跑 + 掉队检测

* 连续监测机器人后方是否有人
* 若人未跟上 → Stop → Turn_in_place → 语音反馈

---

## 3. 语音反馈系统

**优先级：P1**

### 健康模式（轻松）：

* “伸个懒腰～走走啦！”

### Boss 模式（激将法）：

* “再不走就要延毕了！”
* “别怂，我在你前面！”

# 🔧 非功能性要求

| 项目  | 要求                          |
| --- | --------------------------- |
| 安全性 | 距离 < 0.3m 强制急停；速度 ≤ 0.5 m/s |
| 稳定性 | 办公室走廊不能丢定位、不能撞人             |
| 续航  | 一次完整流程耗电 ≤ 10%              |

# 🛠 技术架构

```
硬件：Unitree Go1-Edu + 思岚 A1 LiDAR + Jetson Nano  
系统：Ubuntu 18.04 + ROS Melodic  
算法：Gmapping + AMCL + Move_base + TEB  
应用：Python Flask (Web UI) + rospy
```

# 📏 验收标准（怎么算完成？）

1. **建图成功**：办公室地图清晰，无重影
2. **导航成功**：点击按钮 → 狗自动走到目的地
3. **掉队检测成功**：人停下 → 狗 3 秒内做出反应
4. **完整闭环演示成功**：
   “接单 → 领跑 → 掉队恢复 → 到达 → 待命”
